# Support

There is no commercial support offering for universal-spawn. What
follows is the support ladder for the open standard.

## Ask in this order

1. **GitHub Discussions** —
   [`github.com/AgentMindCloud/universal-spawn/discussions`](https://github.com/AgentMindCloud/universal-spawn/discussions)
   is the default channel for questions, design discussions, RFCs, and
   show-and-tell. Search first; ask second.
2. **GitHub Issues** — use only for **defects** (schema bug, broken
   link, wrong example). Questions posted as issues will be converted
   to discussions.
3. **Discord** — the community chat. Real-time but not a substitute
   for Discussions. Link is in the repository sidebar once invites are
   live.

## When each is the right channel

| You want to…                                      | Channel            |
|---------------------------------------------------|---------------------|
| Ask how to express something in a manifest        | Discussions — Q&A |
| Report a schema rejecting a valid manifest        | Issues — bug       |
| Propose a new field or enum value                 | Discussions — RFC  |
| Propose a new platform folder                     | Issues — new_platform form |
| Show off a creation that uses universal-spawn     | Discussions — Show and tell |
| Ask about governance, sponsorship, trademark      | Discussions — Q&A, or email below |
| Report a security vulnerability                   | `security@agentmindcloud.org` — see [SECURITY.md](SECURITY.md) |
| Question the code of conduct                      | `conduct@agentmindcloud.org` — see [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) |

## Response expectations

- Discussions: best-effort, usually within a week.
- Issues: triaged within seven days.
- Security: acknowledgement within 48 hours (see [SECURITY.md](SECURITY.md)).

## What we cannot help with

- Platform-specific implementation problems. If your Claude skill
  won't load, Claude is the right support channel. If your Vercel
  build fails, Vercel is the right support channel.
- Third-party tooling. If your validator disagrees with ours, open an
  issue on the validator.
- Commercial deployments. universal-spawn is a specification; we are
  not a hosted service.
