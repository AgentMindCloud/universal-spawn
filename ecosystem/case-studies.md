# Case studies

Real-world stories of teams who shipped a universal-spawn manifest
and what changed for them.

This page is a structured skeleton. Entries land as PRs once a team
has run the standard in production for a quarter and is willing to
share numbers.

## Template for an entry

```markdown
## <Project name>

**Use case.** <Two sentences.>

**Before universal-spawn.** <Two sentences. What custom config /
README scraping / manual review they did.>

**After universal-spawn.** <Two sentences. What concrete behavior
changed — Spawn-it button, validation in CI, fewer support
tickets, etc.>

**Numbers (optional).** <Anything quantitative the team is willing
to share. Install conversion, support ticket reduction, time to
first deploy, etc.>

**Manifest excerpt.** <Up to 30 lines.>

**Lessons learned.** <Up to 3 bullets.>

**Manifest URL.** <Public link to the manifest.>
```

### How to contribute

1. Run universal-spawn in production for at least a quarter.
2. Open an issue with the title `[case study] <Your team>`.
3. Maintainer responds with the template above; you fill it in.
4. PR lands here.

Maintainers do not gate by company size or industry. The bar is:
the team genuinely runs the standard, the numbers are honest, the
lessons are useful to other readers.
