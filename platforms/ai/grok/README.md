# Grok (xAI) — universal-spawn platform extension

> **This is the universal-spawn companion to
> [`AgentMindCloud/grok-install`](https://github.com/AgentMindCloud/grok-install).
> If you have a working `grok-install.yaml`, add a
> `universal-spawn.yaml` with a `platforms.grok` block to gain
> cross-platform discoverability. Neither file is required to
> remove the other.**

Grok is the hero platform in `platforms/ai/` because the
universal-spawn idea grew out of `grok-install` v2.14. The two files
are designed to cohabit in the same repository: `grok-install.yaml`
keeps its Grok-specific details, and `universal-spawn.yaml` adds the
cross-platform surface. A spawn host that knows only Grok reads
`grok-install.yaml`; any other consumer reads `universal-spawn.yaml`.

This folder describes how xAI Grok consumes a universal-spawn v1.0
manifest. It defines:

- A strict `platforms.grok` extension schema (draft-07) that composes
  with the master v1.0 schema via `allOf`.
- Three worked examples: a minimal tool-calling agent, a full
  Grok-4-powered research agent, and a creative example that uses
  Grok to narrate a generative art feed.
- A field-by-field compatibility map between the universal manifest
  and the platform-specific `grok-spawn.yaml` / `grok-install.yaml`
  forms.
- A perks file listing what Grok could offer manifests that
  advertise `platforms.grok` — priority discovery, deploy badges,
  billing tie-ins.

## Discovery aliasing

A repository that ships Grok support may contain any (or all) of:

1. `grok-install.yaml` — the prior platform-specific manifest. Still
   valid. `grok-install` v2.14 consumers read it verbatim.
2. `grok-spawn.yaml` — the platform-specific form of
   universal-spawn targeted at Grok. Canonically generated by
   lowering from the universal manifest; useful when a consumer
   wants to see just the Grok view.
3. `universal-spawn.yaml` with a `platforms.grok` block — the
   cross-platform form. Any universal-spawn consumer reads it, Grok
   included.

The discovery order a Grok consumer **SHOULD** follow:

```text
universal-spawn.yaml (with platforms.grok)
  → grok-spawn.yaml
  → grok-install.yaml
```

The first match wins. Tooling **MAY** warn when multiple forms exist
so authors can choose which to keep.

## What Grok cares about

Grok's spawn host is, for practical purposes:

- An API key scope (`XAI_API_KEY`) that owns a billing account.
- A rate limiter scoped to that key.
- A cost ceiling also scoped to that key.
- A tool registry that gets populated from the manifest's
  `tool-call` entrypoints.
- A system prompt loaded from `platforms.grok.system_prompt_file`
  when present.

Grok does **not** care about `visuals.*`, `metadata.maintainers`,
or `data_residency` today; those are surfaced in the card UI but do
not gate spawning.

## What platform-specific extras unlock

- `model` — pin a specific Grok model (`grok-4`, `grok-4-heavy`,
  `grok-4-fast`). Consumers respect the pin when the calling key has
  access.
- `tools[]` — function tools that get registered ahead of the first
  user turn. The `function_ref` is a relative path to a JSON file
  following xAI's tool-use schema.
- `surface[]` — which Grok surfaces may spawn the manifest:
  `grok-api`, `grok-x-integration`, `grok-compute`.
- `streaming` — `true` when the creation expects server-sent events
  back. Purely informational in v1.0.
- `real_time_data` — `true` when the creation relies on Grok's live
  X data signals. Consumers may refuse if the key lacks the
  entitlement.

## How the universal manifest maps to Grok

| Universal field                       | Grok use                                                  |
|---------------------------------------|------------------------------------------------------------|
| `version`                             | Required literal `"1.0"`.                                 |
| `name`, `description`, `summary`      | Card text.                                                |
| `type` ∈ {`ai-agent`, `ai-skill`}     | Accepted directly.                                        |
| `safety.min_permissions`              | Mapped to outbound-host allowlist on the tool runtime.    |
| `safety.rate_limit_qps`               | Enforced at the key scope.                                |
| `safety.cost_limit_usd_daily`         | Enforced as a daily spend cap on the key.                 |
| `safety.safe_for_auto_spawn`          | Controls the first-run confirmation gate.                 |
| `env_vars_required`                   | Surfaced at install; secrets go to the xAI credential store. |
| `metadata.source.commit`              | Pinned in the installed record for audit.                 |
| `platforms.grok`                      | Strictly validated by `grok-spawn.schema.json`.           |
| `x-ext.org.agentmindcloud.grok-install` | Preserved verbatim for the grok-install lowering tool. |

## Minimum viable Grok-only manifest

```yaml
version: "1.0"
name: Ping Grok
type: ai-agent
description: Minimal Grok tool-calling agent.
platforms:
  grok:
    model: grok-4-fast
    surface: [grok-api]
    tools:
      - name: ping
        function_ref: tools/ping.json
env_vars_required:
  - name: XAI_API_KEY
    description: xAI API key.
    secret: true
safety:
  min_permissions: [network:outbound:api.x.ai, model:call:grok-4-fast]
  safe_for_auto_spawn: true
metadata:
  license: Apache-2.0
  author: { name: You, handle: yourhandle }
  source: { type: git, url: https://github.com/yourhandle/ping-grok }
```

See [`grok-spawn.yaml`](./grok-spawn.yaml) for the annotated full
template and [`examples/`](./examples) for three complete manifests.

## Compatibility table

| Manifest field            | Enforces on Grok? | Notes                                          |
|---------------------------|:-----------------:|-------------------------------------------------|
| `version`                 | required          | MUST be `"1.0"`.                               |
| `name`, `description`     | used              | Shown in the xAI console's installed list.     |
| `type`                    | required          | Accepts `ai-agent`, `ai-skill`, `workflow`, `cli-tool`. |
| `summary`                 | used              | Compact card.                                  |
| `safety.min_permissions`  | enforced          | Sandbox allowlist.                             |
| `safety.rate_limit_qps`   | enforced          | Per-key rate limiter.                          |
| `safety.cost_limit_usd_daily` | enforced      | Per-key daily cap.                             |
| `safety.safe_for_auto_spawn` | enforced       | First-run prompt.                              |
| `safety.data_residency`   | used              | Routing hint; default ignored until xAI opens multi-region. |
| `deployment.*`            | used              | Informational for external registries.         |
| `env_vars_required`       | enforced          | Missing required secret blocks spawn.          |
| `metadata.*`              | used              | Card / audit trail.                            |
| `visuals.*`               | used              | Card imagery.                                  |
| `platforms.grok`          | enforced          | Strict validation.                             |
| Other `platforms.*`       | ignored           | Grok doesn't read sibling platform blocks.     |
| `x-ext.*`                 | preserved         | Not interpreted.                               |

See [`compatibility.md`](./compatibility.md) for the detailed map,
including the `grok-install` v2.14 round-trip.
